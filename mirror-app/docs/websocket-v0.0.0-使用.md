# 版本 3（ws）· 安装包 · 安装与使用 ★推荐真机版

**一句话**：真机安装 `reflexion-v3-ws.apk`，手机**直连 Qwen 云**，**全双工实时**和 Aria 对话——**不需要任何服务器**（key 已内置，联网即用）。这是智能镜真机部署的目标形态。

- 安装包：`dist-apks/reflexion-v3-ws.apk`（约 52MB，arm64-v8a，Android 7.0+，debug 签名）
- 对话方式：**全双工流式**——直接自然说话，无需按键，VAD 自动断句，Aria 近实时回应、可打断。
- 音频：原生 PCM 流（`expo-pcm-audio`：AudioRecord 16k 采集 / AudioTrack 24k 播放）。
- **必须真机**：Android 模拟器的虚拟麦克风 HAL 无法送音（`pcm_readi I/O error`），代码没问题，但模拟器听不到你说话。

---

## 一、安装

### USB + adb（推荐）
```bash
adb install -r dist-apks/reflexion-v3-ws.apk
```
出现 `Success` 即成功（手机需先开「开发者选项 → USB 调试」；连点"版本号"7 次解锁开发者选项）。

### 侧载（无电脑）
把 `reflexion-v3-ws.apk` 传到手机 → 文件管理器点开 → 允许"安装未知应用" → 安装。被 Play Protect 拦就选"仍要安装"。

> ⚠️ 与 v2 包名相同（`com.reflexion.mirror`），**不能与 v2 同时安装**，后装的会覆盖先装的。要装 v3 而机上有 v2：`adb uninstall com.reflexion.mirror` 后再装。

### 授权
首次进对话，全部允许：🎤 **麦克风**（必需）、📷 **相机**（多模态筛查用）。

---

## 二、进入"每日检查"

1. 打开 App，等启动自检跑完。
2. 没配后台 → 停在**配对页**。**不用配对**，直接点金色按钮 **「开始每日检查(演示)→」**。
3. 进入 **「Reflexion 检查 · 测试」**。顶部"版本"应显示 **ws**。

> 建议先点 **「🔧 硬件自检」** 确认：`configuredMode = ws`，且显示 **"原生 PCM 流模块已加载"**（绿色）。这说明音频链路就绪。

---

## 三、对话（v3 的体验：直接说）

1. 点 **「开始对话」**，授权麦克风/相机。
2. 状态变为 **Listening**，Aria 先开场。
3. **直接自然说话即可**——不用按任何键。你停顿时 Aria 就会回应，可以打断它、像真人对话一样往返。
4. 按 Aria 引导走完 4 个阶段。
5. 结束：
   - Aria 说出告别语 → 短暂等音频放完 → **自动结束并生成判断**；
   - 或随时点 **「结束并评估」** 手动结束。
6. 出判断卡片：分级、风险分、发现、支持/反对风险证据、**视觉观察**。

## 四、排错

| 现象 | 原因 / 处理 |
|------|-------------|
| 硬件自检显示"未加载 PCM 模块" | 装成了非 v3 包，或架构不符。确认 `configuredMode=ws`、是 arm64 真机。 |
| 首次连接报"检查 token/区域" | 常是首连瞬态；点结束再"开始对话"重连一次即可（正常会 Listening）。 |
| 一直 Listening、说话没反应 | ①确认麦克风权限已给；②确认网络能到 `dashscope.aliyuncs.com`（中国区实时接口）；③在**真机**上测（模拟器麦克风不工作）。 |
| 听不到 Aria 声音 | 检查手机媒体音量；确认不是静音/勿扰；模拟器同样无法播放。 |
| 装不上 | 需 arm64 真机、Android 7+；若机上有 v2 先卸载同包名。 |

## 五、真机 vs 这次在 Mac 上验证到哪一步

本次在 Mac 的 Android 模拟器上已验证：v3 原生包能装能跑、`expo-pcm-audio` 模块加载、直连 Qwen 的 WebSocket 能连上、摄像头预览+采帧正常、编排（Aria 开场/推进）正常、麦克风采集代码正常打开（AudioRecord、RECORD_AUDIO 已授权）。**唯一无法在模拟器验证的是真实音频收发**（模拟器 HAL 限制）——这正是需要在**真机**上跑这个 APK 来确认的部分。

---

## 六、从"自测包"到"正式产品"

这个 APK 为方便自测把 key 内置在了包里（`EXPO_PUBLIC_ALLOW_INSECURE_CLIENT_KEY=true`）。正式上线时应改为：设备不带原始 key，改向一个**云端** token 签发接口（`/api/qwen-token`）领取短期 token 再连 Qwen。这样设备侧依旧**不需要本地服务器**，只需联网 + 一个云端接口。

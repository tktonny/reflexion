# Reflexion Mirror · 三个版本 · 设备测试总览

三个版本是**同一个 App 的三种"对话引擎"**，由构建时的环境变量 `EXPO_PUBLIC_CONVERSATION_MODE` 决定，界面/筛查逻辑完全一样，只有"怎么和 Aria 说话"不同。

| 版本 | mode | 交付形态 | 说话方式 | 需要服务器吗 | 适用平台 |
|------|------|----------|----------|--------------|----------|
| **v1** relay | `relay` | 网页(Chrome) | 全双工流式(直接说) | **要**中转服务器 `npm run relay` | 桌面 Chrome（音频引擎只支持 web） |
| **v2** http | `http` | 安装包 `reflexion-v2-http.apk` | 按住说话(回合制) | 不要（key 已内置） | 真机 Android（也能在模拟器跑） |
| **v3** ws | `ws` | 安装包 `reflexion-v3-ws.apk` | 全双工流式(直接说) | 不要（直连 Qwen，key 已内置） | **真机 Android**（模拟器麦克风 HAL 不工作） |

> 对应的详细文档：
> - `docs/v1-relay-使用.md`
> - `docs/v2-http-apk-使用.md`
> - `docs/v3-ws-apk-使用.md`

## 为什么 v1 没有安装包，v3 却可以直连？

Qwen 的实时 WebSocket 要求把鉴权放在 HTTP **请求头**里（`Authorization: Bearer …`）。

- **浏览器不能给 WebSocket 设置请求头** → 所以 v1(网页)必须在旁边放一个"中转服务器"来补上鉴权。这就是 relay 存在的唯一原因。
- **安卓原生可以设置请求头** → 所以 v3 让手机**直接连 Qwen 云**，中间**不需要任何服务器**。

```
v1(浏览器):  浏览器 ──✗不能加头── [中转服务器] ── Qwen 云
v3(原生App):  安卓App ────直接WS+请求头──────────── Qwen 云
```

所以真机上用 **v3**，身边不需要服务器；relay 只是浏览器的补丁。

## 两个 APK 的共同点

- **独立运行**：JS 已打进包里，Qwen key 已内置（`EXPO_PUBLIC_ALLOW_INSECURE_CLIENT_KEY=true`），**不需要后台、不需要配对**，联网即可自测。
- **架构**：`arm64-v8a`（2017 年后的手机基本都是；不支持 x86 模拟器和 32 位老设备）。
- **系统**：Android 7.0+（Expo SDK 56）。
- **签名**：用的是 **debug 签名**（不是正式发布签名）。安装时系统会提示"来自未知来源/未经检测"，属正常，选"仍要安装"。
- **包名相同**：两个包都是 `com.reflexion.mirror`，**不能同时装**——装了 v3 再装 v2 会互相覆盖（视为升级）。要同时装两个，需要我改成不同包名再重编（可随时找我做）。

## 安装 APK（两种方式，任选其一）

### 方式 A：USB + adb（推荐，最快）
1. 手机打开「设置 → 开发者选项 → USB 调试」（开发者选项：连点"版本号"7 次解锁）。
2. USB 连电脑，手机上弹窗选"允许 USB 调试"。
3. 电脑执行：
   ```bash
   adb install -r dist-apks/reflexion-v3-ws.apk      # 或 reflexion-v2-http.apk
   ```
   `-r` = 覆盖安装。看到 `Success` 即成功。

### 方式 B：手机侧载（无需电脑）
1. 把 `.apk` 传到手机（微信文件/网盘/USB 拷贝）。
2. 用文件管理器点开 `.apk`。
3. 系统提示"允许安装未知应用" → 打开权限 → 安装。
4. 若被 Play Protect 拦截，选"仍要安装/详情 → 仍要安装"。

### 安装后：授权
首次进对话会请求权限，**全部允许**：
- 🎤 **麦克风**（必需——不给就没法说话）
- 📷 **相机**（用于多模态筛查采集面部帧；不给则只做语言筛查）

## 打开后怎么进到"每日检查"（两个 APK 一样）

APK 没有配后台，所以启动后会走到**配对页**（"This mirror is not linked yet"）。**不用去配对**——

1. 启动 App，等自检跑完（几秒）。
2. 到配对页后，点最下面的金色按钮 **「开始每日检查(演示)→」**。
3. 进入 **「Reflexion 检查 · 测试」** 界面（demo 病人，无需后台）。
4. 点 **「开始对话」**，授权麦克风/相机，Aria 会先开场。
5. 按各版本的方式和 Aria 对话（见各自文档）。
6. Aria 说完告别语会**自动结束并出判断**；也可随时点 **「结束并评估」** 手动结束。
7. 结果卡片包含：分级、风险分、发现、支持/反对风险的证据、**视觉观察**（参与度/情绪/警觉，非诊断）。

> 想确认装的是哪个版本 + 硬件是否就绪：进对话界面点 **「🔧 硬件自检」**，会显示当前 `configuredMode`（relay/http/ws）和原生 PCM 模块是否加载。
